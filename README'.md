# 🚀 SigmaTrack — Система управления проектами и задачами (Project Tracker)

Современный высокопроизводительный таск-трекер. Проект спроектирован по принципам **Clean Architecture** и **DDD (Domain-Driven Design)**. Разделен на два независимых слоя: клиентское приложение на **Nuxt 4** и бэкенд ASP.NET web api на **.NET 10**.

---

## 🏗️ Архитектура системы

### ⚙️ Backend (API Layer)

Решение разделено на 4 независимых проекта для обеспечения слабой связанности и строгого контроля зависимостей:

1. **`SigmaTrack.Domain`** (Ядро системы): Чистый C# без внешних зависимостей. Содержит доменные сущности (`Issue`, `Project`, `User`, `Notification`, `IssueLink`), перечисления и бизнес-исключения.
2. **`SigmaTrack.Application`** (Бизнес-логика): Реализован по принципу **Vertical Slices (Features)**. Каждая фича (например, `CreateIssue`) изолирована в своей папке и содержит команды/запросы, хэндлеры (UseCase), DTO и FluentValidation.
3. **`SigmaTrack.Infrastructure`** (Инфраструктура): Реализует интерфейсы доступа к данным (контекст БД EF Core, миграции PostgreSQL) и внешние интеграции.
4. **`SigmaTrack.WebApi`** (Точка входа): Minimal APIs, конфигурация DI-контейнера, JWT-аутентификация, Claims-авторизация, интеграция со Scalar и `GlobalExceptionHandler` (формат RFC 7807).

### 💻 Frontend (Client Layer)

* **Framework:** Nuxt 4 (Vue 3, Composition API) с полной типизацией на **TypeScript**.
* **UI & Styles:** Nuxt UI + Tailwind CSS (Адаптивный дизайн, поддержка темной/светлой темы).
* **State Management:** Pinia (Модульные сторы для управления проектами, задачами и уведомлениями).
* **Internationalization:** Nuxt i18n (Полная локализация интерфейса RU/EN).

## 🛠️ Стек технологий

* **Runtime & СУБД:** .NET 10 (C#) + PostgreSQL
* **ORM & Валидация:** Entity Framework Core + FluentValidation
* **Фронтенд:** Node.js + Nuxt 4 + Tailwind CSS + Pinia

---

## 📌 Реализованный функционал (Базовая версия)

* **Аутентификация и пользователи:** Регистрация, авторизация (JWT + Claims), управление личным профилем, глобальный поиск пользователей.
* **Управление проектами:** Создание, редактирование и удаление проектов; управление участниками и ролями; система инвайтов в команду (принять/отклонить).
* **Работа с задачами (Issues):** Создание, просмотр деталей, редактирование и удаление задач. Автогенерация уникальных номеров задач внутри проекта, назначение исполнителей, Story Points, тегирование и комментарии. Эндпоинт «Активные задачи пользователя» для вывода тасок со всех проектов.
* **Система уведомлений & UI/UX:** Умный роутинг (при переходе по уведомлению из другого проекта система автоматически переключает контекст активного проекта в шапке).

## ⏳ В разработке (Бэклог функционала)

* [ ] **Обзор (Dashboard)** — главное меню проекта со сводной аналитикой, графиками и активностью команды.
* [ ] **Канбан-доска (Kanban Board)** — визуальное управление рабочим процессом и перетаскивание карточек задач по статусам.
* [ ] **Спринты (Sprints)** — планирование итераций, запуск и завершение спринтов, интеграция со Story Points.
* [ ] **Контейнеризация и оркестрация** — разделение компонентов системы по Docker-контейнерам (`docker-compose`).
* [ ] **Reverse Proxy** — внедрение и настройка Nginx для маршрутизации запросов фронтенда/бэкенда и обработки CORS.
* [ ] **Хранилище файлов (File Storage)** — разработка подсистемы хранения для привязки картинок и документов к комментариям, аватаркам пользователей и задачам.

## 🚀 Локальный запуск (Разработка)

1. **Настройка бэкенда**:
* Перейдите в папку `SigmaTrack.WebApi`.
* Скопируйте `appsettings.json` в `appsettings.Development.json` и укажите строку подключения в `ConnectionStrings:DefaultConnection`.
* Примените миграции:
```bash
dotnet ef database update --project SigmaTrack.Infrastructure --startup-project SigmaTrack.WebApi

```

* Запустите API: `dotnet run --project SigmaTrack.WebApi`.

2. **Настройка фронтенда**:
* Установите зависимости через `pnpm install`.
* Запустите локальный сервер разработки: `pnpm dev`.
